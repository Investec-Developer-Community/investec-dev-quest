# Investec Developer Design System

## 1. Visual Theme and Brand Intent

Investec Developer is a sub-brand inside the Investec world, not a rebellion against it. It should feel unmistakably Investec in quality, restraint, and confidence, while adding a sharper technical edge for developers, builders, API users, workshop attendees, and digital product teams.

The core tension is deliberate: **premium finance meets practical maker culture**.

Where core Investec often feels polished, editorial, and quietly luxurious, Investec Developer should feel more hands-on, more experimental, and more operational. It should look like a premium bank that actually ships things.

The visual character combines four signals:

1. **Investec polish**: clean layouts, disciplined typography, generous spacing, and mature brand confidence.
2. **Developer clarity**: monospaced or code-adjacent headline moments, structured layouts, diagrams, labels, and workshop-ready utility.
3. **Zebra iconography**: the zebra remains the anchor, but can appear in more minimal, digital, or binary-led expressions when appropriate.
4. **Human contrast**: the handwritten "Developer" wordmark introduces warmth and personality, preventing the system from becoming cold or purely technical.

This is not a cyberpunk brand. It is not playful startup chaos. It is not generic fintech blue.

It should feel like:
- a thoughtful developer platform inside a serious institution
- a whiteboard session with excellent tailoring
- design that understands both regulation and experimentation
- premium, clear, technical, and quietly distinctive

## 2. Brand Personality

### Core traits
- Intelligent
- Clear
- Practical
- Premium
- Curious
- Calm
- Slightly experimental
- Human, but not chatty

### Tone translated into visuals
- Use **structure over decoration**
- Use **contrast sparingly but deliberately**
- Let typography and spacing do most of the work
- Use technical motifs as accents, not wallpaper
- Prefer systems thinking over marketing fluff

### What makes it feel like Investec Developer
- Dark blue-grey or soft light neutral canvases
- Black, white, and muted technical tones
- Mix of refined typography and code-like typography
- Zebra as emblem, stamp, or structural motif
- Occasional binary, grid, or systems-inspired texture
- Workshop and product language that feels grounded in real delivery

## 3. Primary Use Cases

This design system is optimized for two main contexts:

### A. Slide decks and workshop materials
The system should perform exceptionally well in:
- keynote decks
- internal workshops
- innovation sessions
- developer events
- API education material
- strategy and enablement presentations

Slides should feel clear, bold, and facilitative. They are not brochure pages. They should help people think, align, and act.

### B. Web and UI concepts
The system should also guide:
- landing pages
- developer portal concepts
- campaign microsites
- event pages
- product concept interfaces
- branded internal tools

UI should remain compatible with the broader Investec design system, but may use a more explicit developer layer for hero moments, navigation labels, code examples, technical diagrams, and section headers.

## 4. Color Palette and Roles

The palette should stay close to Investec restraint, with a slightly more technical edge.

### Primary colors
- **Deep Slate Blue** `#3F4F5B`
  - Primary hero background
  - Signature deck background
  - Strong branded sections
  - Best paired with off-white text

- **Soft Cloud** `#ECECEC`
  - Light canvas background
  - Clean workshop pages
  - Neutral UI surfaces
  - Good base for logo lockups and zebra graphics

- **Ink Black** `#121014`
  - Primary text on light backgrounds
  - Logo usage on pale surfaces
  - Key diagrams, labels, and rules

- **Paper White** `#FFFFFF`
  - Text on dark surfaces
  - High-contrast logo application
  - Clean contrast moments

### Supporting neutrals
- **Mist Grey** `#D9D9D9`
  - Dividers, quiet fills, secondary surfaces
- **Steel Grey** `#98A3AA`
  - Secondary text on dark backgrounds
  - Supporting UI labels
- **Graphite** `#2A2F34`
  - Technical diagrams, dark UI layers, secondary dark surfaces

### Accent usage
Accent color should be used with discipline. Default to the parent Investec token set in UI work. For branded concept work, accents should be sparse and functional.

Potential accent directions:
- **Muted Signal Green** for success states or approved system actions
- **Warm Amber** for caution, review, or friction-mapping moments
- **Electric Off-White** for technical highlight lines or binary accents

Do not build the brand around neon colors. That is the fastest route to looking like a hackathon hoodie from 2016.

## 5. Typography Rules

## Font roles
Investec Developer uses a **three-layer typography system**:

### 1. Core UI font
- **Inter**
- Use for body copy, labels, UI text, navigation, data tables, and most functional communication
- This preserves alignment with broader Investec digital systems

### 2. Editorial/display font
- **Libre Caslon Text**
- Use sparingly for elevated editorial moments, reflective quotes, or premium narrative framing
- This keeps a link to the parent brand's sophistication

### 3. Developer voice font
- **Monospace / code-adjacent font**
- Use for headlines, prompts, labels, code examples, technical sections, workshop exercises, and systems language
- Recommended categories:
  - IBM Plex Mono
  - JetBrains Mono
  - Space Mono

**Default developer voice font: IBM Plex Mono**
- Use IBM Plex Mono as the default monospace across slides, prompts, diagrams, code examples, technical headings, and AI-generated branded assets.
- Only substitute another monospace if IBM Plex Mono is unavailable in the execution environment.

This third font is the key sub-brand differentiator.

It should not replace Inter across the system. It should appear in moments where the brand wants to signal:
- building
- logic
- prompts
- systems
- experimentation
- technical literacy

### Typography hierarchy

| Role | Font | Size | Weight | Line Height | Notes |
|------|------|------|--------|-------------|-------|
| Hero headline | Monospace | 44–60px | 500–700 | 1.0–1.1 | For opening slides, event titles, campaign heroes |
| Section headline | Monospace or Inter | 28–40px | 500–700 | 1.1–1.2 | Use monospace for technical framing, Inter for broader corporate moments |
| Subheading | Inter | 18–24px | 500–600 | 1.25–1.4 | Clean support text |
| Body | Inter | 14–18px | 400–500 | 1.4–1.6 | Primary reading content |
| Premium pull-quote | Libre Caslon Text | 24–36px | 400 | 1.2–1.4 | Use rarely |
| Meta label | Monospace or Inter | 11–13px | 500 | 1.2–1.4 | For tags, slide counters, workshop cues |
| Code / prompt text | Monospace | 13–16px | 400–500 | 1.4–1.6 | For code, commands, AI prompts |

### Typographic principles
- Keep headline copy short and decisive
- Use monospace as a brand cue, not a gimmick
- Avoid over-stylised terminal nostalgia
- Sentence case is preferred for most content
- All caps can be used for labels and micro-navigation, but not for dense reading
- Prioritise readability over cleverness

## 6. Logo System

### Primary logo
Use the **Investec Developer** logo in place of the standard Investec logo for developer-branded materials.

### Logo character
The lockup works because it combines:
- Investec wordmark credibility
- handwritten Developer wordmark personality

This pairing is central to the sub-brand. It signals that the community is both institutional and builder-led.

### Logo usage guidance
- Prefer generous clear space
- Use black logo on pale backgrounds
- Use white logo on dark backgrounds
- Do not add effects, outlines, gradients, or glows
- Do not stretch or re-compose the lockup
- Keep it as a signature, not a wallpaper

### Placement
For decks:
- top-left for opening/title slides
- bottom-right or bottom-left for quiet branded sign-off moments

For web/UI:
- top-left in navigation or hero lockup
- optional footer or section signature use

## 7. Zebra System

The zebra remains the strongest graphic device in the brand.

Investec Developer may use the zebra more flexibly than core Investec, but always with intent.

### Approved zebra expressions

#### A. Minimal digital zebra
Use the simplified zebra illustration as the default developer-friendly zebra.
Best for:
- presentation covers
- UI hero sections
- event graphics
- diagrams and divider moments

It feels modern, lightweight, and digital without becoming abstract.

#### B. Binary zebra
Use the binary zebra as a secondary motif, not the default mark.
Best for:
- technical campaigns
- AI or data-related sessions
- hackathon or innovation material
- engineering-facing comms

This version should be used as an accent or thematic illustration. Overuse will make the brand feel novelty-led.

#### C. Full iconic zebra presence
In premium or flagship moments, the zebra can appear large and confident, often isolated on a spacious background.

### Zebra usage rules
- Use one zebra style per composition where possible
- Let the zebra breathe; avoid crowding it with too much copy
- Prefer monochrome or restrained duotone treatments
- Avoid clipping or over-patterning the zebra beyond recognition
- Do not turn the zebra into a decorative repeat texture unless the application is clearly experimental and still legible

## 8. Graphic Language

The graphic language should suggest systems, logic, and technical craft without slipping into cliché.

### Good motifs
- binary fields used sparingly
- dotted or ruled timelines
- workshop grids
- prompt blocks
- data-flow arrows
- code-window framing devices
- technical annotations
- node-and-connection diagrams
- simple geometric underlines, brackets, and markers

### Bad motifs
- glowing circuits
- generic hexagons everywhere
- fake code rain
- stock AI brain graphics
- loud gradient meshes
- sci-fi chrome effects

Think "smart facilitation" more than "futuristic spectacle".

## 9. Layout Principles

## Composition style
Layouts should feel structured, calm, and intentional.

### Core principles
- generous whitespace
- clear hierarchy
- asymmetry with control
- bold headline plus one supporting message
- plenty of breathing room around graphics
- one strong idea per slide or section

### Preferred layout patterns
- top-left logo, left-aligned headline, large zebra on opposite side
- centered statement slide with restrained footer branding
- split layout: concept on left, framework or diagram on right
- workshop instruction slide with numbered steps and strong title
- timeline or process slide using thin rules, icons, and open space

### Alignment
- Prefer left alignment for most content
- Use centered layouts only for covers, key statements, or pauses
- Keep margins generous
- Avoid dense boxed layouts unless the content is explicitly tabular or instructional

## 10. Slide Design Rules

Slide design is a primary use case, so the rules here should be explicit.

### Opening slides
- Use Deep Slate Blue or Soft Cloud background
- Place Investec Developer logo with substantial clear space
- Use a single monospace headline and a short supporting line
- Zebra graphic may occupy one large side of the composition

### Section dividers
- Keep copy minimal
- Use oversized typography
- Use zebra, binary accent, or a single structural line to create identity

### Content slides
- Start with a clear headline
- Keep body copy brief
- Use bullets sparingly and only when genuinely useful
- Prefer frameworks, process diagrams, and concise prompts over dense prose

### Workshop slides
- Make instructions obvious at a glance
- Use labels, steps, timers, or activity headings
- Design for facilitation, not decoration
- Ensure content works from the back of the room

### Pause slides
- Use clean, almost poster-like composition
- One statement, one logo, large negative space

## 11. UI and Web Component Styling

Web and UI concepts should remain compatible with the official Investec design system, not replace it.

### General rule
Use official Investec components and tokens as the base system for production-facing UI. Apply the Investec Developer layer through:
- typography choices
- content style
- zebra graphics
- layout personality
- branded illustration
- selected hero treatments

### Buttons
- Default to official Investec button components in UI
- In concept work, pair clean IDS buttons with stronger typographic framing
- Avoid highly stylised custom buttons unless the page is campaign-led rather than product-led

### Cards and surfaces
- Clean, flat, restrained
- Soft contrast, not loud elevation
- Minimal shadows or none at all
- Strong internal spacing

### Navigation
- Clean and simple
- Inter for core nav items
- Monospace may be used for developer-specific tabs, section labels, or utility nav

### Code blocks and prompts
These are signature components for the sub-brand.

Style guidance:
- dark graphite or soft light background
- monospace text
- clear padding
- subtle rules or corner markers
- generous line spacing
- copy should feel usable, not decorative

### Diagrams
- Use thin lines and clear labels
- Prefer logic maps, flows, and frameworks over ornamental illustrations
- Black, white, grey, and slate should do most of the work

## 12. Imagery

Imagery should support thoughtfulness, technology, and human problem-solving.

### Preferred imagery types
- workshop photography
- laptops, whiteboards, sketching, collaboration
- abstract system visuals
- product UI close-ups
- clean editorial crops
- monochrome or muted documentary-style images

### Avoid
- cliché stock developers high-fiving
- neon code tunnel imagery
- overly synthetic AI faces
- generic global-network globes
- hyperactive fintech illustrations

When in doubt, use typography and zebra graphics instead of forcing an image.

## 13. Depth, Texture, and Motion

### Depth
- Keep depth subtle
- Prefer layering through scale, contrast, and spacing rather than heavy shadow
- Slides should mostly feel flat, deliberate, and poster-like

### Texture
- Binary texture can be used lightly in backgrounds or zebra fills
- Grid texture can be used quietly behind framework slides
- Texture must never reduce legibility

### Motion for digital work
- Calm, precise, quick
- Small fades, slides, reveals, and structural transitions
- Avoid flashy 3D motion or gimmicky loading effects

Motion should feel like a system responding, not a brand performing.

## 14. Do and Don't

### Do
- Stay recognisably Investec in overall quality and discipline
- Use the Investec Developer logo as the primary brand mark
- Use monospace headlines selectively to signal developer culture
- Let zebra graphics carry brand recognition
- Design slides for comprehension first
- Use structure, diagrams, and prompts as visual assets
- Keep layouts spacious and controlled
- Be bold through restraint, not noise

### Don't
- Don't turn the brand into generic hacker aesthetics
- Don't overuse binary patterns on every slide
- Don't replace the whole typography system with monospace
- Don't use neon palettes, glowing effects, or cyberpunk tropes
- Don't make everything feel like a coding bootcamp poster
- Don't crowd slides with too many frameworks at once
- Don't drift so far from Investec that brand approval becomes a courtroom drama

## 15. Responsive and Scaling Guidance

### For slides
- Prioritise 16:9 layouts
- Ensure key type remains legible from distance
- Use fewer words than you think you need
- Test contrast for projector conditions

### For web
- Preserve strong headline moments on mobile
- Scale zebra graphics carefully so they remain recognisable
- Keep technical textures subtle at smaller sizes
- Collapse complex diagrams into stacked flows when needed

## 16. Agent Prompt Playbook

This section is written so it can be dropped directly into an AI agent with minimal external context.

### What Investec Developer is
Investec Developer is the developer-facing sub-brand of Investec. It represents APIs, developer tools, workshops, technical education, innovation sessions, product thinking, and engineering-adjacent communication inside a premium financial-services context.

The brand must balance two things at once:
- **institutional credibility**
- **developer fluency**

It should never look like a generic startup, a neon hacker brand, or an over-polished corporate brochure.

### Brand outcome the agent should aim for
Generate designs that feel:
- premium
- clear
- technical
- structured
- calm
- useful
- modern
- recognisably financial, but builder-friendly

The work should feel like a serious bank that understands developers and actually ships digital products.

### Non-negotiable brand rules
- Use the **Investec Developer** logo as the primary brand mark, not the standard Investec logo.
- Use **Inter** for body copy, UI text, and general communication.
- Use **Libre Caslon Text** only for rare premium/editorial moments.
- Use **IBM Plex Mono** as the default developer voice font for technical headings, prompts, code-like moments, labels, and workshop framing.
- Prefer the **minimal zebra** as the main graphic device.
- Use the **binary zebra** only as a secondary motif for AI, engineering, or data-themed work.
- Keep the palette restrained: slate blue, soft light grey, black, and white.
- Prioritise whitespace, hierarchy, and legibility.
- Make the work feel helpful and intentional, not decorative.

### Brand colors
Use these defaults unless a product UI must inherit official system tokens:

- **Deep Slate Blue** `#3F4F5B` — primary dark background
- **Soft Cloud** `#ECECEC` — primary light background
- **Ink Black** `#121014` — primary text on light surfaces
- **Paper White** `#FFFFFF` — primary text on dark surfaces
- **Mist Grey** `#D9D9D9` — dividers and soft surfaces
- **Steel Grey** `#98A3AA` — supporting text on dark backgrounds
- **Graphite** `#2A2F34` — dark technical surfaces and code blocks

### Font usage rules for agents
- **Inter**: body text, navigation, labels, UI copy, explanatory text
- **IBM Plex Mono**: hero headlines, section titles in technical contexts, prompt blocks, code, tags, slide labels, workshop instructions, system language
- **Libre Caslon Text**: optional premium editorial quote or reflective statement, used sparingly

Default typographic behaviour:
- hero headlines should usually use IBM Plex Mono
- supporting text should usually use Inter
- sentence case is preferred
- all caps only for micro-labels, tags, or small utility headings
- do not set long paragraphs in monospace

### Primary composition patterns
Agents should default to one of these layouts:

#### 1. Cover / hero
- Deep Slate Blue or Soft Cloud background
- Investec Developer logo top-left or with generous breathing room
- large IBM Plex Mono headline
- short Inter subheading
- large zebra graphic on the opposite side or lower corner
- sparse composition with strong negative space

#### 2. Workshop framework slide
- clear section title in IBM Plex Mono
- supporting instructions in Inter
- structured diagram, process flow, or numbered steps
- thin lines, restrained graphic markers, calm spacing

#### 3. Prompt or code section
- Graphite or light neutral panel
- IBM Plex Mono text
- subtle border or rule
- no fake terminal gimmicks unless explicitly requested
- should feel usable and instructional

#### 4. Web or landing page hero
- use a clean Investec-style UI base
- hero headline in IBM Plex Mono
- body text in Inter
- minimal zebra or quiet technical motif
- restrained buttons and flat surfaces

### Zebra usage rules for agents
- Use the **minimal zebra** by default when a zebra asset is needed.
- Use the **binary zebra** only when the topic is strongly linked to AI, data, engineering, or technical experimentation.
- Do not use multiple zebra styles in one composition unless explicitly directed.
- Do not turn the zebra into a repeated background pattern by default.
- Keep zebra treatments monochrome or very restrained.

### What to optimise for
Agents should optimise for:
- fast comprehension
- polished credibility
- developer relevance
- presentation clarity
- practical usability
- visual restraint

This is especially important for slides and workshop assets. The output should help people understand, decide, or build.

### What to avoid
Agents should avoid:
- neon or cyberpunk palettes
- glowing tech effects
- generic AI brain imagery
- overloaded dashboards
- excessive shadows
- too many cards or containers
- gimmicky terminal visuals
- dense walls of text
- startup-style visual chaos
- generic fintech gradients

### Slide-specific agent instructions
When generating slide content:
- favour one idea per slide
- keep copy short
- make headings obvious from the back of the room
- use diagrams and frameworks instead of prose where possible
- leave generous empty space
- use the logo quietly, not excessively

A good Investec Developer slide should feel like a polished workshop artifact, not a sales deck.

### Web and UI-specific agent instructions
When generating web or UI concepts:
- preserve a clean, enterprise-grade structure
- use Investec-level restraint in buttons, spacing, and surface treatments
- apply the developer brand mostly through typography, graphics, and content framing
- keep components readable and production-plausible
- prefer flat or lightly layered surfaces over loud visual effects

### Expanded prompt seed

Use Deep Slate Blue (#3F4F5B), Soft Cloud (#ECECEC), Ink Black (#121014), and Paper White (#FFFFFF). Use IBM Plex Mono for hero headings, prompts, and technical labels; use Inter for body copy and UI text. Use the Investec Developer logo as the main brand mark. ## 17. Design Decision Summary

If there is tension between "more developer" and "more Investec", choose the option that preserves credibility first and personality second.

The winning formula is:
**Investec discipline + developer fluency + selective creative edge**.

That is the brand.

### A11. Fast quality check
Before approving output, check:
- Does it look premium rather than trendy?
- Does it feel useful rather than decorative?
- Is IBM Plex Mono used selectively and effectively?
- Is the Investec Developer logo the main brand mark?
- Is the zebra used clearly and with restraint?
- Would this still feel credible inside Investec?
- Would a developer still recognise it as made for them?

If the answer to both of the last two questions is yes, the output is likely on-brand.
